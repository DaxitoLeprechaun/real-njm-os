## Design Context

### Users
Consultants and the CEO at Disrupt — a strategic consulting agency. They supervise autonomous AI agents (PM Agents, CEO Agent) that generate deliverables, run strategic frameworks, and escalate blockers. The interface is used in focused, heads-down work sessions. Users are technically literate, information-hungry, and allergic to fluff. They want to see the signal, not the decoration.

### Brand Personality
**Three words:** Dense. Authoritative. Precise.

NJM OS is a control plane, not a chatbot. It should feel like a tool that knows more than you do and presents that knowledge without softening it. The tone is analytical, Spanish-language, and direct — no marketing copy, no friendly onboarding metaphors. Operators use this to make decisions that have real consequences.

### Aesthetic Direction
**Reference:** Bloomberg Terminal meets Linear — information density without chaos, high contrast without gimmicks, every pixel serving a function.

**Theme:** Dark. This is a heads-down work environment. Operators run this on a monitor in an agency context.

**Palette — OKLCH-based, tinted monochrome:**
- Surfaces: Near-black tinted very slightly warm (no pure #000000). Background: `oklch(0.09 0.005 60)`. Cards/panels: `oklch(0.13 0.006 60)`.
- Text: High-contrast off-white `oklch(0.93 0.005 60)`, secondary `oklch(0.55 0.006 60)`.
- Success (LISTO PARA FIRMA): Raw neon-adjacent green — `oklch(0.72 0.22 145)` — used functionally, never decoratively.
- Danger (BLOQUEO CEO, "Escudo del CEO"): Blood red — `oklch(0.50 0.22 25)` — confrontational, not softened.
- Borders: `oklch(0.22 0.005 60)` solid 1px. No glow, no blur.

**Typography:**
- UI chrome and labels: `Geist` (or `Geist Mono` for IDs, timestamps, transaction codes).
- Agent output text (propuesta, justificación): A modern serif — `Spectral` or `Libre Baskerville` — to visually separate human-readable AI prose from UI scaffolding.
- Monospace data: `Roboto Mono` or `Geist Mono` for error logs, file paths, IDs.

**Anti-references:** No landing page aesthetics. No soft card shadows. No gradient fills of any kind. No glassmorphism. No rounded-corner cards with gentle drop shadows — this is not a SaaS onboarding flow.

### Design Principles

1. **Function over form, but form IS function.** Visual hierarchy should make the operator's decision path obvious: what happened → what it means → what to do next. Every visual choice earns its place by serving cognition.

2. **Hard edges, high contrast.** Use 1px solid borders instead of shadows. Use `box-shadow: 2px 2px 0 #000` hard-offset when shadow is truly needed — never `blur > 0`. This is a brutalist-corporate tool.

3. **Functional color as signal, not decoration.** Green and red are status indicators. They appear only where status matters. They do not appear in backgrounds, headers, or decorative elements outside their semantic context.

4. **Information density without noise.** Labels, IDs, timestamps, and metadata are always visible — not hidden behind hovers. Operators need the full picture at a glance. Use compact spacing with intentional rhythm, not generous padding.

5. **Agent output reads differently than UI chrome.** Prose generated by agents (proposals, justifications, error logs) uses a serif typeface to signal: "this came from a mind, not a template." UI chrome (labels, buttons, nav) stays monospace or grotesque.
